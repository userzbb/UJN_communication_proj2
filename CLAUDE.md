# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Digital voice transmission system implemented in Simulink — BPSK and D-BPSK modulation over AWGN channel with PCM encoding/decoding and RRC pulse shaping, targeting Xilinx Zynq-7020 FPGA via HDL Coder and Vivado.

## Critical: Model Name Collision Between BPSK and D-BPSK

`models/bpsk/` and `models/dbpsk/` contain files with the **same names** (`voice_tx_system.slx`, `voice_tx_hdl_cosim.slx`). MATLAB resolves models by name, not path. **Always remove the unused variant from path before loading the other:**

```matlab
% Before working with D-BPSK models:
rmpath('models/bpsk');
cd('models/dbpsk');
load_system('voice_tx_system');

% Before working with BPSK models:
rmpath('models/dbpsk');
addpath('models/bpsk');
load_system('voice_tx_system');
```

## Signal Chain Architecture

```
Audio Source (.wav) → PCM Encoder (Quantizer → Int2Bit) → BPSK Modulator (BPSK Mod → RRC Tx Filter) → AWGN Channel → BPSK Demodulator (RRC Rx Filter → BPSK Demod) → PCM Decoder (Bit2Int → Dequantizer) → Audio Output
```

The HDL-cosim models substitute fixed-point implementations for HDL Coder compatibility.

## HDL Architecture (BPSK)

**Fixed-point format**: `sfix16_En14` — 16-bit signed, 14 fractional bits, range [-2, +2), resolution ~6.1e-5.

**LUT mapping** in `BPSK_Mod_HDL`:
- Bit 0 → `+1.0` = `0x4000` (16384)
- Bit 1 → `-1.0` = `0xC000` (-16384)

**Pipeline** (2 clock cycles, posedge clk):
1. `BPSK_Mod_HDL`: Input bit sampled at posedge N → LUT → init_guard mux → pipeline register → output valid at posedge N+2
2. `BPSK_Demod_HDL` (`CmpZero`): Combinational — `input < 0 → 1`, `input ≥ 0 → 0`

**Total loopback latency**: 2 clock cycles. bits_out[C] corresponds to bits_in[C-2].

## HDL Architecture (D-BPSK)

Same LUT mapping as BPSK. Differential encoder placed between bit_in and LUT; differential decoder placed between DTConv and bit_out.

**D-BPSK Algorithm**:
- Encoder: `enc[n] = bit[n] XOR enc[n-1]` (init=0)
- Decoder: `bit[n] = enc[n] XOR enc[n-1]`

**DBPSK_Mod_HDL** internal structure: `bit_in → Diff_Enc_XOR → LUT → [Delay?] → sym_out` with `Diff_Enc_Reg` (Unit Delay, init=0) feeding XOR input 2.

**DBPSK_Demod_HDL** internal structure: `rx_sym_in → CmpZero → DTConv → Diff_Dec_XOR → bit_out` with `Diff_Dec_Reg` feeding XOR input 2.

**Pipeline** (3 clock cycles):
1. DBPSK_Mod_HDL: bit_in → XOR → LUT → pipeline register → sym_out (2 cycles)
2. DBPSK_Demod_HDL: CmpZero → DTConv → XOR + feedback reg → bit_out (1 cycle)

**Total loopback latency**: 3 clock cycles. bits_out[C] = bits_in[C-3].

## Common Commands

```matlab
% First, add project directories to MATLAB path (from project root):
addpath('models/bpsk', 'models/dbpsk', 'scripts/bpsk', 'scripts/dbpsk', 'scripts/common', 'scripts/common/board_def');

% Open and run the floating-point reference model
open_system('voice_tx_system');
sim('voice_tx_system');

% Open and run the HDL-cosim model
open_system('voice_tx_hdl_cosim');
sim('voice_tx_hdl_cosim');

% Run the BPSK HDL cosimulation
run('scripts/bpsk/create_bpsk_hdl_cosim.m');

% BER vs Eb/N0 sweep
run('scripts/common/run_ber_sweep.m');

% Play the reference audio
[audio, fs] = audioread('audio/voice_input.wav');
sound(audio, fs);
```
